# 04 错误码与恢复策略表

> 本文档定义全系统统一的 ErrorCode、ErrorLevel、ErrorSource 枚举，以及每个 error 默认的 RecoveryAction。所有节点的状态实现必须从此枚举中选取，不允许临时编号。

---

## 0. 设计原则

```text
1. error_code 是 uint16，全局唯一，不允许复用
2. 编号区间按 source 划分，便于一眼看出错误来自哪
3. 每个 error 必须配 level + source + 默认 recovery
4. 状态代码报错时只填 error_code，level 和 recovery 由 ErrorHandler 查表得出
5. 同一个错误现象在不同节点出现时，复用同一个 error_code（不要按节点重新编号）
6. error_code = 0 永远表示 "无错误"，禁止占用
```

---

## 1. ErrorLevel

```python
class ErrorLevel(IntEnum):
    INFO  = 0    # 正常事件记录，不影响流程
    WARN  = 1    # 自动恢复的小异常（一次重试通常能过）
    ERROR = 2    # 需要主动恢复（重感知/重规划/换目标）
    FATAL = 3    # 需要人工介入或中止任务
    ESTOP = 4    # 必须立即急停
```

**Level 决定 FSM 的反应强度**：
- INFO/WARN：状态自己处理，不上报上层
- ERROR：交给本层的 RecoveryFSM 处理
- FATAL：上报到 TaskFSM，进入 FAIL_TASK 或人工恢复
- ESTOP：直接触发 SystemFSM 的 E_STOP 状态

## 4. RecoveryAction

```python
class RecoveryAction(IntEnum):
    NONE                     = 0    # 不做恢复，仅记录
    RETRY_CURRENT_STATE      = 1    # 当前状态重试一次
    REPLAN                   = 2    # 重新规划（用于运动规划失败）
    REPERCEPTION             = 3    # 重新开窗采样/重新融合感知 Topic
    SWITCH_TARGET            = 4    # 换下一个候选目标（pair）
    SWITCH_PHASE             = 5    # 跳到另一个 phase
    MOVE_BASE                = 6    # 调整底盘位置后重试
    RETREAT_SAFE             = 7    # 撤回到安全位
    RELOCALIZE               = 8    # 重新定位
    REBUILD_GRID             = 9    # 重新建网格
    WAIT_MANUAL_RECOVERY     = 10   # 等待人工恢复
    ABORT_TASK               = 11   # 中止当前任务
    E_STOP                   = 12   # 立即急停
```

## 7. error→recovery 映射的覆盖机制

每个 error 表里有 **default_recovery**。但具体业务可以在状态层级覆盖：

```python
# 状态可以重写本状态遇到某 error 时的 recovery
class RunPairSelection(BaseState):
    custom_recovery = {
        ErrorCode.E_PAIR_NO_REACHABLE: RecoveryAction.MOVE_BASE,   # 默认就是这个，写出来便于阅读
        ErrorCode.E_PAIR_DUAL_CONFLICT: RecoveryAction.SWITCH_TARGET,
    }
```

ErrorHandler 处理顺序：
```text
1. 看状态 custom_recovery 里有没有
2. 没有则用 error_codes.yaml 里现场可调的覆盖表
3. 都没有则用本文档的 default_recovery
```

## 8. 重试上限

每种 RecoveryAction 有默认的 max_attempts：

| RecoveryAction | 默认 max_attempts | 备注 |
|---|---|---|
| RETRY_CURRENT_STATE | 3 | 同状态最多重试 3 次 |
| REPLAN | 2 | |
| REPERCEPTION | 3 | |
| SWITCH_TARGET | 5 | 至多换 5 次候选 |
| MOVE_BASE | 2 | |
| RETREAT_SAFE | 1 | 不重试，撤回失败直接 FATAL |
| RELOCALIZE | 2 | |
| REBUILD_GRID | 2 | |
| ABORT_TASK / E_STOP / WAIT_MANUAL_RECOVERY | 1 | 终态 |

超过上限自动升级为 FATAL（recovery=WAIT_MANUAL_RECOVERY）。

具体值在 `error_codes.yaml` 现场可调。

```text

on_failure / on_timeout 返回 StateResult(error_code=X)
  ↓
FSMEngine 写日志、广播 /fsm/error
  ↓
本层 ErrorHandler 查表
  ├── INFO/WARN：本状态自己处理或忽略
  ├── ERROR：本层 RecoveryFSM 处理（如 WallRecoveryFSM）
  ├── FATAL：上报到 TaskFSM → FAIL_TASK 或 WAIT_MANUAL_RECOVERY
  └── ESTOP：触发 SystemFSM 的 E_STOP 状态，所有 Action goal 立即 cancel
```

## 10. 实现要求

### 10.1 fsm_core/error_code.py

```python
from dataclasses import dataclass
from enum import IntEnum

class ErrorLevel(IntEnum):
    INFO  = 0
    WARN  = 1
    ERROR = 2
    FATAL = 3
    ESTOP = 4

class ErrorSource(IntEnum):
    SYSTEM = 0
    # ... 见 §2

class ErrorCode(IntEnum):
    NO_ERROR                = 0
    E_SYS_BOOT_FAIL         = 1000
    E_SYS_HARDWARE_NOT_READY= 1001
    # ... 全量
    E_GRASP_UNKNOWN         = 5099
    # ... 全量

@dataclass(frozen=True)
class ErrorMeta:
    code: int
    name: str
    level: ErrorLevel
    source: ErrorSource
    default_recovery: 'RecoveryAction'
    description: str

ERROR_TABLE: dict[int, ErrorMeta] = {
    1000: ErrorMeta(1000, "E_SYS_BOOT_FAIL", ErrorLevel.FATAL,
                    ErrorSource.SYSTEM, RecoveryAction.WAIT_MANUAL_RECOVERY,
                    "启动自检失败"),
    # ... 全量
}

def get_error_meta(code: int) -> ErrorMeta:
    return ERROR_TABLE.get(code, ERROR_TABLE[1099])  # 兜底
```

**强约束**：
- ERROR_TABLE 与本文档 §5 必须完全一致；任何变更先改文档再改代码
- 单测必须覆盖：每个 code 都能查到 meta、meta.code 与 dict key 一致

### 10.2 配置覆盖文件 error_codes.yaml

```yaml
# error_codes.yaml（现场可调）
overrides:
  # 例如：现场发现真空漏气重试一次没用，直接切目标
  5102:
    recovery: SWITCH_TARGET
    max_attempts: 1

  # 现场关掉某些误报告警
  3221:
    level: INFO

max_attempts:
  RETRY_CURRENT_STATE: 3
  REPLAN: 2
  # ...
```

ErrorHandler 启动时加载 overrides，运行时优先使用。

RUN_PAIR_SELECTION 状态：
