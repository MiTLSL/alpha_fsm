# 09 任务拆解与开发计划

> 本文档把 FSM 设计落地拆成可执行的开发任务清单，含负责人、依赖、阻塞关系与里程碑。**总原则：先冻 fsm_msgs 与 fsm_core，再 mock-first 跑通策略闭环，最后逐步替换真实后端**。
>
> 配套阅读：
> - 01 系统架构（节点划分与归属）
> - 03 接口契约（fsm_msgs 字段定型）
> - 04 错误码（先冻 enum 再开发）
> - 07 Mock 与测试（每个任务验收用例）
> - `codes/IK、视觉雷达感知与底盘导航代码导览.md` §10（P0/P1/P2 决策清单）

---

## 0. 拆分原则

```text
1. 任务粒度：每条任务 1-3 天可独立完成、可独立验收（不堆积大块）。
2. 验收标准：每条任务必须给出"完成看哪个测试用例"的引用（07 文档 ID）。
3. 依赖显式：用 → 表示阻塞；冻结期内不允许下游开始改上游字段。
4. 平行化：除 fsm_core / fsm_msgs 外，其余尽量平行；A/B 与你三人三路推进。
5. P0 才是冻结期阻塞。P1 在主线跑通后并入；P2 可滞后，必要时绕过。
6. 任务里所有日期均为绝对日期（基准日 2026-05-23）。
```
