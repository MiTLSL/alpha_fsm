# 05 配置与参数清单

> 本文档列出 5 个 YAML 文件的全部参数，含默认值、单位、可调范围、生效节点。**现场调参主要动 `business.yaml`，其他文件开发期定型后较少改动**。

---

## 0. 加载流程

```text
launch 文件加载 YAML
  → 注入到对应节点的 ROS2 parameters
  → 节点 __init__ 时 declare_parameter() + get_parameter()
  → 缓存到 ctx.config（@dataclass(frozen=True)）
  → 状态代码读 ctx.config.xxx，禁止直接调 get_parameter()
```

支持运行时 reload：`ros2 service call /<node>/reload_config std_srvs/Trigger`，由 node.py 实现，重新读 YAML 并替换 ctx.config。

**调参指引**：
- 现场新部署，先标定 `observation_pose / left_phase_workpose / right_phase_workpose / fixed_place_pose`，其他保留默认
- 视觉漏检多 → 调小 `match_tolerance` 或加 `mapping_window.frames`
- 视觉误检多 → 调高 `detection_filter.confidence_min`
- 抓取失败多 → 调大 `max_retry_per_slot`，或检查 workspace 设定是否过宽
- 任务过早完成（其实还有箱）→ 加大 `empty_confirm_frames`

## 4. params/error_codes.yaml（累积式增加）

> error 默认 recovery 与等级的现场覆盖。`overrides` 段优先于 ErrorCode enum 默认。

```yaml
# params/error_codes.yaml
error_codes:
  # —— 覆盖默认 level / recovery ——
  # 例如：现场实测发现真空漏气重试基本无效，直接换目标
  # 5102:
  #   recovery: "SWITCH_TARGET"
  #   max_attempts: 1
  overrides: {}

  # —— 现场静音某些 WARN（保留日志但不进 recovery 流程）——
  # 例如：误检过多但不影响业务
  # silenced:
  #   - 3221
  silenced: []

  # —— Recovery 全局上限（覆盖 fsm.yaml 中的同名设置）——
  # 留空表示用 fsm.yaml 的默认
  recovery_max_attempts: {}

  # —— FATAL 升级阈值 ——
  # 当某 error_code 在同一 task 内累计触发次数超过这个值，自动升级 FATAL
  escalate_to_fatal:
    5102: 5                                  # vacuum leakage 5 次就 fatal
    4022: 3                                  # navigation stuck 3 次就 fatal
```

## 7. 配置访问 Python 模板

每个节点用同一套读法：

```python
# wall_destacking_strategy/wall_destacking_strategy/node.py
from dataclasses import dataclass

@dataclass(frozen=True)
class BusinessConfig:
    box_size_l: float
    box_size_w: float
    box_size_h: float
    match_tol_y: float
    match_tol_z: float
    empty_confirm_frames: int
    max_retry_per_slot: int
    allow_single_arm_grasp: bool
    fixed_place_pose: tuple
    # ... 其他字段
    @classmethod
    def from_ros_params(cls, node: rclpy.node.Node) -> "BusinessConfig":
        return cls(
            box_size_l = node.declare_parameter("business.box_size.length", 0.40).value,
            box_size_w = node.declare_parameter("business.box_size.width",  0.40).value,
            box_size_h = node.declare_parameter("business.box_size.height", 0.40).value,
            match_tol_y = node.declare_parameter("business.match_tolerance.y", 0.15).value,
            match_tol_z = node.declare_parameter("business.match_tolerance.z", 0.15).value,
            empty_confirm_frames = node.declare_parameter("business.empty_confirm_frames", 5).value,
            max_retry_per_slot = node.declare_parameter("business.max_retry_per_slot", 3).value,
            allow_single_arm_grasp = node.declare_parameter("business.allow_single_arm_grasp", True).value,
            fixed_place_pose = (
                node.declare_parameter("business.fixed_place_pose.x", 0.5).value,
                node.declare_parameter("business.fixed_place_pose.y", 0.0).value,
                node.declare_parameter("business.fixed_place_pose.z", 0.8).value,
                # ... qx/qy/qz/qw
            ),
            # ...
        )
```

**强约束**：
- 每个节点必须在 `__init__` 中一次性把所有需要的参数读入 `ctx.config`
- 状态代码读 `ctx.config.xxx`，**绝不**在状态内 `self.get_parameter()`
- `BusinessConfig` 必须 `@dataclass(frozen=True)`，运行时若需 reload，由 `/reload_config` 触发完整重建

## 8. 参数版本与变更管理

```yaml
# 每个 YAML 文件顶部加版本号注释
# version: 1.0.0
# last_updated: 2026-05-23
# changelog:
#   1.0.0 初版
```

变更规则：
- **patch 升级**（1.0.x）：现场调值，不影响其他模块
- **minor 升级**（1.x.0）：新增字段（带默认值，向后兼容）
- **major 升级**（x.0.0）：字段重命名/删除，需同步改代码
